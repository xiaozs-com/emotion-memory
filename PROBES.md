# PROBES.md — 情感探测机制

> 主动情感探测：基于心理学量表，定期主动发起对话式测量。

## 核心设计

### 探测类型

| 类型 | 触发频率 | 题量 | 用途 |
|------|---------|------|------|
| **情绪温度** | 每周日 | 5题 | 快速捕捉本周情绪状态 |
| **量表探测** | 每季度 | 20-36题 | 用标准化量表深度测量 |
| **即时探测** | 随时 | 1-3题 | 对话中自然插入 |

### 探测时机

1. **定时探测**：每周日早上 9:00 自动发起（`emotion_schedule.py`）
2. **对话触发**：对话中出现情绪信号时自然插入
3. **主动发起**：累计 7 天无情感数据时提醒

### 结果处理流程

```
外部量表完成 → 用户转发结果 → 解析 → 存储到 scales/*.yaml
→ 分析得分 → 更新 profile.md → 更新 insights.md
→ 下次对话时应用
```

### 量表数据存储格式

每个量表一个 YAML 文件，包含：
- `meta`: 量表名称、版本、测量维度
- `history`: 历次测量结果（带时间戳）
- `latest`: 最新结果和维度得分

### 与外部平台 xinlishi.qxq.chat 的对接

外部平台完成量表施测后，用户将结果发给我处理：

**支持的结果格式：**
1. 自然语言描述（如："我做了ECR-R，依恋焦虑得分38，依恋回避得分22"）
2. 截图/图片 → OCR 识别
3. 文本报告复制粘贴

**数据字段规范（xinlishi 对接时使用）：**
```yaml
scale: ecr36
date: "2026-04-15"
raw_scores:
  anxiety: 38
  avoidance: 22
interpretation:
  anxiety_level: "中等焦虑型"
  avoidance_level: "低回避型"
  attachment_style: "安全型"
```

## 量表清单

### 核心量表（必测）

| 量表 | 维度 | 题数 | 测量内容 |
|------|------|------|---------|
| ECR-R | 焦虑/回避 | 36 | 成人依恋风格 |
| BFI-16 | 五大人格 | 16 | 人格特质 |
| PHQ-4 | 抑郁/焦虑 | 4 | 心理健康筛查 |
| SWLS | 生活满意度 | 5 | 主观幸福感 |

### 辅助量表（选测）

| 量表 | 维度 | 题数 | 测量内容 |
|------|------|------|---------|
| PANAS | 正性/负性情感 | 20 | 情绪状态 |
| RSE | 自尊 | 10 | 自我价值感 |
| 情绪温度计 | 当前情绪 | 1 | 快速情绪捕捉 |

## 使用方式

### 手动触发探测
```bash
python emotion_probe.py mood        # 情绪温度（5题）
python emotion_probe.py scale ecr36 # ECR-R 依恋量表（36题）
python emotion_probe.py quick       # 快速探测（1题）
```

### 定时任务
```bash
python emotion_schedule.py setup    # 设置每周日定时探测
python emotion_schedule.py check    # 检查未完成探测
```

### 接收外部量表结果
```
用户：这是我今天的ECR-R结果：焦虑38分，回避22分
AI：  收到，正在分析...
     → 存储到 scales/ecr36.yaml
     → 更新 profile.md 依恋风格标签
     → 记录到 insights.md
     → 解读：你的依恋风格为[安全型]，这意味着...
```

## 隐私说明

- 所有数据仅存储在本地 `memory/emotion/scales/` 目录
- 不上传任何外部服务
- 用户可随时查看、修改、删除个人数据
